# 登录认证使用说明

## 📚 概述

项目使用JWT Token进行身份认证，**基于路径前缀自动判断**，用户和护工的上下文信息分开管理。

---

## 🎯 路径规则

### 自动认证规则

```
/api/user/login         ✅ 公开（用户登录）
/api/user/register      ✅ 公开（用户注册）
/api/user/*             🔒 需要用户身份，自动存入 UserContextHolder

/api/caregiver/login    ✅ 公开（护工登录）
/api/caregiver/register ✅ 公开（护工注册）
/api/caregiver/*        🔒 需要护工身份，自动存入 CaregiverContextHolder

/api/file/*             🔒 需要登录（用户或护工都可以）

/api/health             ✅ 公开（健康检查）
/api/doc.html           ✅ 公开（接口文档）
```

## 💻 在业务代码中获取当前用户

### 用户端接口 - 使用 `UserContextHolder`

```java
@RestController
@RequestMapping("/user")
public class UserOrderController {

    @Resource
    private OrderService orderService;

    // 创建订单
    @PostMapping("/order")
    public Result<Void> createOrder(@RequestBody OrderDTO dto) {
        // 获取当前登录用户ID
        Long userId = UserContextHolder.getUserId();
        String username = UserContextHolder.getUsername();
        
        orderService.createOrder(userId, dto);
        return Result.success();
    }

    // 查询我的订单
    @GetMapping("/orders")
    public Result<List<Order>> getMyOrders() {
        Long userId = UserContextHolder.getUserId();
        List<Order> orders = orderService.getByUserId(userId);
        return Result.success(orders);
    }

    // 更新个人信息
    @PutMapping("/profile")
    public Result<Void> updateProfile(@RequestBody ProfileDTO dto) {
        Long userId = UserContextHolder.getUserId();
        userService.updateProfile(userId, dto);
        return Result.success();
    }
}
```

### 护工端接口 - 使用 `CaregiverContextHolder`


## 🔐 认证流程

### 1. 用户端登录
```http
POST /api/user/login
Content-Type: application/json

{
  "username": "zhangsan",
  "password": "123456"
}

# 返回
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "zhangsan",

  }
```

## 🎁 核心优势

### ✅ 零注解开发
```java
// ❌ 不需要这样
@RequireLogin
@GetMapping("/info")
public Result<User> getInfo() { ... }

// ✅ 直接这样
@GetMapping("/info")
public Result<User> getInfo() {
    Long userId = UserContextHolder.getUserId();
    ...
}
```

### ✅ 上下文分离
```java
// 用户端用 UserContextHolder
Long userId = UserContextHolder.getUserId();

// 护工端用 CaregiverContextHolder  
Long caregiverId = CaregiverContextHolder.getCaregiverId();
```

### ✅ 自动类型校验
- 用户Token访问 `/api/user/*` ✅
- 用户Token访问 `/api/caregiver/*` ❌ 403错误
- 护工Token访问 `/api/caregiver/*` ✅
- 护工Token访问 `/api/user/*` ❌ 403错误

## 📖 完整开发示例

### 用户端 - 订单管理

```java
@RestController
@RequestMapping("/user")
public class UserOrderController {

    @Resource
    private OrderService orderService;

    // 创建订单（自动获取当前用户ID）
    @PostMapping("/order")
    public Result<Void> createOrder(@RequestBody CreateOrderDTO dto) {
        Long userId = UserContextHolder.getUserId();
        
        Order order = new Order();
        order.setUserId(userId);
        order.setServiceType(dto.getServiceType());
        order.setAddress(dto.getAddress());
        
        orderService.save(order);
        return Result.success("订单创建成功");
    }

    // 查询我的订单列表
    @GetMapping("/orders")
    public Result<List<Order>> getMyOrders() {
        Long userId = UserContextHolder.getUserId();
        return Result.success(orderService.getByUserId(userId));
    }

    // 取消订单
    @PostMapping("/order/{orderId}/cancel")
    public Result<Void> cancelOrder(@PathVariable Long orderId) {
        Long userId = UserContextHolder.getUserId();
        orderService.cancelOrder(orderId, userId);
        return Result.success();
    }
}
```

## 📱 前端对接

### 用户端小程序/APP

```javascript
// 1. 用户登录
const loginRes = await http.post('/api/user/login', {
  username: 'zhangsan',
  password: '123456'
})

// 2. 保存token
const token = loginRes.data.token
wx.setStorageSync('token', token)

// 3. 后续所有请求都带上token
const config = {
  header: {
    'token': wx.getStorageSync('token')
  }
}

// 4. 访问用户端接口
await http.get('/api/user/orders', config)
await http.post('/api/user/order', orderData, config)
```

## 🎯 核心总结

### 开发时
1. **用户端接口**：写在 `/user/` 下，用 `UserContextHolder.getUserId()`
2. **护工端接口**：写在 `/caregiver/` 下，用 `CaregiverContextHolder.getCaregiverId()`
3. **不需要注解**：路径自动决定认证规则
4. **不需要传ID**：从上下文自动获取

### 运行时
1. **自动拦截**：除了登录注册，其他接口自动需要Token
2. **自动校验**：用户Token不能访问护工接口，反之亦然
3. **自动注入**：用户信息自动注入到对应的ContextHolder
4. **自动清理**：请求结束后自动清理上下文

简洁、清晰、安全！🎉
