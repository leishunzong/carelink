# 护联 Care-Link 前后端联调说明

## ✅ 已完成的工作

### 1. 后端服务信息
- **后端地址**: `http://localhost:8080/api`
- **数据库**: MySQL，本地默认 `localhost:3306`
- **Redis**: 本地默认 `localhost:6379`
- **API 文档**: http://localhost:8080/api/doc.html

### 2. 前端开发配置

#### 代理配置 (`vite.config.ts`)
```typescript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
    }
  }
}
```

#### Axios 请求配置 (`src/utils/request.ts`)
- **Base URL**: `/api`
- **请求头认证**: `Authorization: Bearer <token>`
- **超时时间**: 10秒
- **统一响应处理**: code !== 200 时显示错误

### 3. 已对接的 API 接口

#### ✅ 认证模块 (`src/api/auth.ts`)
- `POST /admin/login` - 管理员登录
- 登录响应: `{ token, userType: 'admin' }`

#### ✅ 管理员模块 (`src/api/admin.ts`)

**入驻审核**
- `GET /admin/caregiver/settle-list` - 获取入驻申请列表
- `POST /admin/caregiver/settle/audit` - 入驻审核（通过/拒绝）

**技能审核**
- `GET /admin/caregiver/skill-apply-list` - 获取技能申请列表
- `POST /admin/caregiver/skill-apply/audit` - 技能申请审核

**护工管理**
- `GET /admin/caregiver/page` - 分页查询护工列表

**订单管理**
- `GET /admin/order/page` - 分页查询订单列表

**评价管理**
- `GET /admin/review/page` - 分页查询评价列表

**标签管理**
- `GET /tag/list` - 获取所有评价标签
- `GET /tag/list/{type}` - 按类型获取标签（1好评/2差评）
- `POST /admin/tag` - 新增标签
- `PUT /admin/tag/{id}` - 修改标签

**技能管理**
- `GET /skill/page` - 分页查询技能列表
- `POST /admin/skill` - 新增技能
- `PUT /admin/skill/{id}` - 修改技能
- `DELETE /admin/skill/{id}` - 删除技能

**服务包管理**
- `GET /admin/package/page` - 分页查询服务包
- `POST /admin/package` - 新增服务包
- `PUT /admin/package/{id}` - 修改服务包
- `DELETE /admin/package/{id}` - 删除服务包
- `PUT /admin/package/{id}/on-shelf` - 上架服务包
- `PUT /admin/package/{id}/off-shelf` - 下架服务包

**RAG 知识库**
- `GET /admin/rag/documents` - 获取文档列表
- `POST /admin/rag/document` - 上传文档
- `POST /admin/rag/document/text` - 提交文本
- `DELETE /admin/rag/document/{id}` - 删除文档

## 🐛 常见问题

### Q1: 跨域问题
**现象**: 浏览器控制台报 CORS 错误  
**解决**: 已在 `vite.config.ts` 中配置代理，前端请求会自动转发到后端

### Q2: 401 未授权
**现象**: 所有请求都返回 401  
**解决**: 
1. 检查是否已登录并获取 token
2. 检查 localStorage 中是否有 token
3. 检查后端 JWT 配置是否正确

### Q3: 图片无法显示
**现象**: 审核材料图片加载失败  
**解决**: 
1. 检查后端返回的图片 URL 是否正确
2. 检查腾讯云 COS 配置是否正常
3. 检查图片 URL 是否包含完整路径

### Q4: Token 过期
**现象**: 请求一段时间后返回 401  
**解决**: 
1. 检查 JWT 过期时间设置
2. 实现 token 自动刷新机制
3. 或在 401 时自动跳转到登录页
